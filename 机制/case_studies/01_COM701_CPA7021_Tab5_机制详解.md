# Case 01：COM701 / CPA.7.021 / Tab5 机制详解

更新时间：2026-07-07

## 0. 这一个案例为什么先看

COM701 / CPA.7.021 / Tab5 是 PVRIG 赛题里最值得先拆的真实案例，因为它同时连接了 5 件事：

```text
临床候选 COM701
    ↓
官方阳性参考 Tab5
    ↓
专利/论文中反复出现的 CPA.7.021
    ↓
PVRIG-PVRL2 blocking 实验证据
    ↓
R95 / I97 / S67 表位线索
```

所以它不是单纯“一个抗体名字”，而是我们理解 PVRIG blocker 的第一把尺子。

当前结论先说清楚：

> COM701/CPA.7.021/Tab5 这条路线的成功，不在于它只是能结合 PVRIG，而在于它能占据 PVRIG 上会影响 PVRL2 结合的功能表位；它告诉我们后续模型必须区分 binder 和 blocker。

---

## 1. 分子身份：COM701、CPA.7.021、Tab5 之间是什么关系

### 1.1 COM701

COM701 是 Compugen 开发的 anti-PVRIG antibody。NCI Drug Dictionary 对它的机制描述非常直接：COM701 结合 CTL 和 NK 细胞上的 PVRIG，阻断 PVRIG 与 PVRL2/CD112 的相互作用，从而解除 PVRIG 诱导的 T/NK 细胞抑制。

Compugen 官网把 COM701 描述为 lead candidate 和 potentially first-in-class immunotherapy，靶向 PVRIG；同时强调它来自 Compugen 的计算发现平台，并且作为单药和与 PD-1 blocker 联用时有初步抗肿瘤活性和安全性信号。

这里要注意措辞：

```text
COM701 是临床阶段成功案例/机制成功案例，
不是已经获批上市的成功药物。
```

### 1.2 CPA.7.021

IBI352g4a 论文把 COM701 作为 benchmark（BMK）时写到：COM701 是 humanized IgG4 PVRIG antibody，并使用来自 CPA.7.021 的重链/轻链序列，相关专利为 US 10,227,408 B2。

这给我们一个很重要的链条：

```text
CPA.7.021
    ↓ humanized / IgG4 format
COM701 benchmark
    ↓ official/public positive family
Tab5 类阳性参考
```

也就是说，CPA.7.021 不是随便一个专利编号，而是理解 COM701/Tab5 这条路线的关键桥梁。

### 1.3 Tab5

大会官网把 Tab5 作为 IgG 阳性参照，并提供 VH/VL 序列。对我们来说，Tab5 的作用主要有两个：

```text
1. positive control：证明这一类 CDR/表位确实有效；
2. leakage exclusion：后续候选 CDR 不能与它过像。
```

不要把 Tab5 当作要模仿的模板。它更像“不能撞车的成功边界”。

---

## 2. 它解决了 PVRIG 机制里的哪一个关键问题

PVRIG 是 inhibitory receptor，PVRL2/CD112 是 ligand。PVRIG-PVRL2 结合后，T/NK 细胞功能被压低。

成功抗体要打断的是这条链：

```text
PVRL2 on tumor/APC
    ↓ binds
PVRIG on CTL/NK
    ↓ inhibitory signal
T/NK activation drops
```

COM701/CPA.7.021/Tab5 这类抗体要实现的是：

```text
antibody binds PVRIG ECD
    ↓
occupies/perturbs functional epitope
    ↓
PVRL2 cannot productively bind PVRIG
    ↓
PVRIG inhibitory signal is reduced
    ↓
T/NK cell function is restored, especially in combination contexts
```

这也是为什么官网复筛看 Kd 和 IC50，而不是只看一个 docking score：

```text
Kd = 是否真的结合 PVRIG
IC50 = 是否真的阻断 PVRIG-PVRL2
```

---

## 3. 关键实验证据一：binder 不等于 blocker

Compugen 专利里的 hybridoma panel 非常有价值。它不是只拿一个抗体做故事，而是筛了一组 anti-PVRIG antibodies。

缓存到的专利文本显示：在 cellular-based biochemical assays 里，29 个 hybridoma antibodies 中有：

```text
20 个 clear blockers
9 个 non-blockers
```

这句话对我们后续模型非常关键：

```text
能结合 PVRIG 的抗体，未必能阻断 PVRIG-PVRL2。
```

所以第一阶段如果我们只训练或排序 `PVRIG binder score`，方向会跑偏。我们真正需要的是：

```text
blocking-oriented binder
```

也就是：结合位置、结合角度、空间位阻都要能干扰 PVRL2。

---

## 4. 关键实验证据二：CPA.7.021 是强 blocker

同一个专利还比较了 CPA.7.021、CPA.7.002、CPA.7.005、CPA.7.050 等抗体在不同 assay orientation 下的阻断能力。

其中一个非常重要的观察是：

```text
CPA.7.021 remained the best blocking antibody.
```

更重要的是，专利说明不同抗体在不同竞争实验方向下的 IC50 会变化；这说明阻断不是一个抽象属性，而依赖真实竞争条件：

```text
抗体先占 PVRIG？
PVRIG 和 PVRL2 同时出现？
抗体是否能在配体竞争下稳定占住功能表位？
```

所以后续模型不能只问：

```text
这个抗体 dock 到 PVRIG 分数高不高？
```

而要问：

```text
当 PVRL2 也要贴上来时，这个抗体是否还能抢占/遮挡/改变 PVRIG 的 functional interface？
```

---

## 5. 关键实验证据三：PVRIG-PVRL2 是非冗余轴，适合联用

Whelan et al. 2019 的论文给 COM701/CPA.7.021 这条路线提供了生物学背景。论文显示：

```text
1. PVRIG-PVRL2 是区别于 TIGIT-PVR 的 nonredundant inhibitory node；
2. PVRIG blockade 可以增强 CD8+ T cell cytokine production 和 cytotoxic activity；
3. PVRIG blockade 与 TIGIT 或 PD-1 blockade 联合时，可进一步增强 T cell activation；
4. 在某些 tumor context 中，PVRL2 表达模式支持 PVRIG 作为 dominant inhibitory pathway。
```

这解释了为什么 COM701 经常和 PD-1 / TIGIT 组合放在一起看。

这也提醒我们：

```text
PVRIG 单抗的 blocking activity 是必要条件；
但真正临床/体内效果可能依赖 tumor context 和 combination context。
```

## 6. 表位线索：S67 / R95 / I97 到底怎么理解

### 6.1 专利里的 epitope mapping

Compugen 专利用 human/cynomolgus PVRIG 差异突变做 epitope mapping。缓存文本显示，S67、R95、I97 会影响部分 anti-human PVRIG antibodies 的 cynomolgus cross-reactivity。

其中：

```text
Group 1：R95/I97 相关，代表包括 CPA.7.021 和 CPA.7.028
Group 2：S67 + R95/I97 相关，代表包括 CPA.7.002 和 CPA.7.041
Group 3：不依赖 S67 或 R95/I97，代表包括 CPA.7.024 和 CPA.7.050
```

注意：专利 OCR/网页文本里有时把 I97 识别成 `197`，这里按 PVRIG 位点解释为 I97。

### 6.2 和我们结构图谱对上以后，优先级变了

我们自己的 `PVRIG_hotspot_set_v1.csv` 已经把这些位点映射到 8X6B / 9E6Y：

| 位点 | 结构状态 | 当前解释 |
| --- | --- | --- |
| R95 | 8X6B 和 9E6Y 都支持的 consensus interface residue | 高权重 soft hotspot |
| I97 | 8X6B 支持，9E6Y 在 4.5 A cutoff 下不支持 | 低/中权重 soft hint |
| S67 | 两个结构都不在 4.5 A interface baseline | 只保留表位线索，不驱动阻断评分 |

R95 在两个结构中都有直接 PVRL2 接触：

```text
8X6B: PVRIG B57 / UniProt R95 -> PVRL2 TYR34, PHE115
9E6Y: PVRIG A55 / UniProt R95 -> PVRL2 TYR64, PHE145
```

I97 的结构支持弱一些：

```text
8X6B: PVRIG B59 / UniProt I97 -> PVRL2 PHE115
9E6Y: PVRIG A57 / UniProt I97 -> no 4.5 A contact
```

S67 当前不在 PVRIG-PVRL2 distance interface：

```text
8X6B: PVRIG B29 / UniProt S67 -> no 4.5 A contact
9E6Y: PVRIG A27 / UniProt S67 -> no 4.5 A contact
```

因此，COM701/CPA.7.021/Tab5 给我们的不是“必须打 S67/R95/I97 三个位点”，而是：

```text
R95/I97 周边确实是已知 blocker 可以利用的功能表位区域；
其中 R95 最值得优先建模；
S67 只能作为某些抗体/cyno cross-reactivity 的表位线索。
```

## 7. 为什么 COM701 选择 Fc-reduced / IgG4 很值得注意

Compugen 官网把 COM701 写成 Fc-reduced anti-PVRIG antibody。IBI352g4a 论文也把 COM701/BMK 描述为 humanized IgG4 PVRIG antibody。

这说明 COM701 的设计可能更偏向：

```text
阻断 PVRIG-PVRL2 inhibitory signaling
而不是强烈依赖 Fc-mediated cell killing
```

但这不是 PVRIG 抗体唯一正确格式。IBI352g4a 论文反而提出 IgG1/Fc-competent format 在 NK cell 场景里可能带来更强体内效果。

所以从 COM701 我们学到的是：

```text
Fc format 是独立设计变量，不能和 epitope blocking 混在一起。
```

后续如果做 VHH，需要提前想清楚：

```text
裸 VHH？
VHH-Fc?
二价 VHH?
TIGIT-PVRIG 双抗？
```

不同 format 可能同样 block PVRIG-PVRL2，但体内功能输出不一样。

## 8. 这个案例给后续 AI 模型的约束

### 8.1 正向特征

后续候选可以因为这些特征加分：

```text
1. 覆盖 PVRIG-PVRL2 consensus interface；
2. 靠近 R95/H92/R98/W100 这片局部表面；
3. 对 PVRL2 贴合方向形成空间位阻；
4. CDR/paratope 与 Tab5/CPA.7.021 不同，但功能上打同一类 interface；
5. 对不同 assay orientation 的竞争仍保持鲁棒性。
```

### 8.2 负向特征

这些需要扣分或过滤：

```text
1. 只结合 PVRIG 背面，远离 PVRL2 interface；
2. 只围绕 S67 设计，牺牲 R95/interface coverage；
3. 与 Tab5/CPA.7.021/HR-151 CDR 高度相似；
4. docking pose 虽好，但不和 PVRL2 产生 steric conflict；
5. 只预测 Kd，不预测 IC50/blocking。
```

### 8.3 对 VHH 设计的启发

VHH 后续不是要“像 Tab5”，而是：

```text
用完全不同的 CDR/paratope，
占据 COM701/CPA.7.021 证明有效的 functional surface，
同时满足 CDR identity < 80% 和 developability。
```

这就是“机制借鉴，不是序列模仿”。

## 10. 本案例的一句话结论

> COM701/CPA.7.021/Tab5 证明了 PVRIG 上存在真实可药物化的 blocking epitope；其中 R95/I97 周边是已知成功 blocker 使用过的功能区域，但后续 VHH 设计不能复制它的 CDR，而应以不同 CDR 占据或干扰同一 PVRIG-PVRL2 functional interface。
