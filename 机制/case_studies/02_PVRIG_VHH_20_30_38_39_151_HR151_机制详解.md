# Case 02：PVRIG-20/30/38/39/151 / HR-151 VHH 机制详解

更新时间：2026-07-07

## 0. 为什么第二个案例看 VHH / 纳米抗体

上一例 COM701 / CPA.7.021 / Tab5 证明了：PVRIG 上确实存在能被抗体阻断的 functional epitope。

第二例要回答的是另一个问题：

```text
PVRIG-PVRL2 是两个 Ig-like domain 的表面贴合，
不是深口袋。
那 VHH / nanobody 这种小型单域抗体，
为什么也能有效阻断这类表面界面？
```

这里最重要的真实案例来自 WO2021180205A1，以及大会官网给出的 HR-151 VHH 阳性参考。

当前结论先说清楚：

> PVRIG-20/30/38/39/151 这一组 VHH 证明了 VHH 可以做到低 nM/sub-nM 级 PVRIG 结合和 nM 级 PVRIG-PVRL2 阻断；其中 151 原始阻断最强，但 151 单药不等于体内最强，真正成功常常来自 VHH-Fc 或 TIGIT-PVRIG 双抗 format。

---

## 1. 分子身份：HR-151 和 PVRIG-151 是什么关系

大会官网把 HR-151 作为 VHH 阳性参照，并给出了 VHH 氨基酸序列。我们本地用官方 validator/ANARCI 提取到的 HR-151 IMGT CDR 是：

```text
CDR1: ASGFTYRPYC      length 10
CDR2: IDIFGGT         length 7
CDR3: AAGDSPDGRCGLPPQGLNY  length 19
```

这个 CDR3 很长，而且含有 cysteine。对 VHH 来说，这类长 CDR3 是常见优势：它可以更深、更突出地进入或覆盖蛋白-蛋白相互作用界面。

但是这里必须非常小心：

```text
HR-151 是官方阳性参照，也是查重对象；
不能围绕 HR-151 做高度相似改造。
```

它更适合作为：

```text
1. positive control；
2. CDR similarity exclusion reference；
3. VHH paratope shape 参考；
4. 长 CDR3 / VHH format 的机制样例。
```

---

## 2. 这不是单个 VHH，而是一组 VHH

WO2021180205A1 不是只公开了一个 151，而是一组 PVRIG binding proteins / single-domain antibodies。

核心原始 VHH 包括：

```text
PVRIG-20
PVRIG-30
PVRIG-38
PVRIG-39
PVRIG-151
```

它们被进一步做成人 IgG4 Fc fusion / full-length anti-PVRIG format，并带有 Fc 区突变：

```text
S228P, F234A, L235A, K447A
```

这说明专利里验证的不是“裸 VHH 小分子”一种形式，而是：

```text
VHH binding domain
    ↓
IgG4 Fc fusion / full-length anti-PVRIG antibody
    ↓
PVRIG binding + PVRL2 blocking + cell activity
```

因此后续我们要区分：

```text
VHH paratope 是否能 block？
VHH 装到 Fc 上以后是否仍然能 block？
VHH 作为双抗模块时是否仍然能 block？
```

## 3. 关键数值：原始 20/30/38/39/151 都能阻断 PVRIG-PVRL2

专利表 7 给出了 human PVRIG/PVRL2 binding 的 ELISA blocking IC50。

| 抗体 | PVRIG/PVRL2 blocking IC50 | 解释 |
| --- | ---: | --- |
| PVRIG-20 | 1.18 nM | 阻断强 |
| PVRIG-30 | 1.11 nM | 阻断强 |
| PVRIG-38 | 0.93 nM | 阻断强 |
| PVRIG-39 | 0.76 nM | 阻断强 |
| PVRIG-151 | 0.37 nM | 这一组里最强 |
| Tab5 | 1.16 nM | IgG 阳性参考，同量级 |
| IgG4 control | no blocking | 阴性对照 |

这个表的机制意义非常大：

```text
1. VHH 可以打 PVRIG-PVRL2 这种表面界面；
2. 151 最强，但 20/30/38/39 也不是弱阳性；
3. 这些 VHH 的 blocking 与 Tab5 在同一数量级；
4. 后续不能只学习 151，要学习多个 VHH 成功形态。
```

## 4. 关键数值：原始 VHH 对 PVRIG 的 Kd 也很强

专利表 8 给出了 anti-PVRIG antibodies 与 human PVRIG 的亲和力。

| 抗体 | Kd | 解释 |
| --- | ---: | --- |
| PVRIG-20 | 2.36E-10 M | 0.236 nM，sub-nM |
| PVRIG-30 | 7.23E-10 M | 0.723 nM，sub-nM |
| PVRIG-38 | 2.17E-9 M | 2.17 nM，低 nM |
| PVRIG-39 | 6.96E-10 M | 0.696 nM，sub-nM |
| PVRIG-151 | 2.00E-10 M | 0.200 nM，sub-nM |
| Tab5 | 2.19E-10 M | 0.219 nM，sub-nM |

这说明：

```text
PVRIG-151 的强 blocking 不是靠弱结合凑出来的，
它本身就是强 binder。
```

但更重要的是：

```text
20、30、39、151 都达到 sub-nM；
38 虽然稍弱，但仍是低 nM。
```

所以在模型里，不应该只把 151 作为唯一阳性中心，而应该把这个系列看成多个可行 VHH solution。

## 5. 人源化/工程化以后，活性仍然能保留

专利后续对 VHH 进行了人源化：将骆驼单域抗体 framework 替换为人 germline heavy-chain framework，保留 CDR，并根据三维结构对埋藏残基、CDR 相互作用残基和构象重要残基做回复突变，同时优化 CDR 区化学不稳定残基。

这一步非常关键，因为它告诉我们：

```text
成功 VHH 不是只靠原始 camelid framework；
framework 可以工程化，但 CDR/结构支撑残基要谨慎处理。
```

专利表 18 中，人源化候选仍有较强 Kd：

| 人源化抗体 | Kd | 解释 |
| --- | ---: | --- |
| 20H5 | 6.98E-11 M | 0.0698 nM，非常强 |
| 30H2 | 1.92E-9 M | 1.92 nM |
| 30H3 | 2.41E-9 M | 2.41 nM |
| 39H1 | 3.28E-9 M | 3.28 nM |
| 39H2 | 6.92E-10 M | 0.692 nM |
| 39H4 | 4.82E-10 M | 0.482 nM |
| 151H7 | 1.20E-9 M | 1.20 nM |

这里有两个重要洞察：

```text
1. 人源化会改变亲和力，不是所有版本都比原始更强；
2. 20H5 反而非常强，说明 151 不是唯一值得研究的 VHH family。
```

## 7. 为什么 VHH 能打 PVRIG-PVRL2 这种界面

PVRIG-PVRL2 是表面贴合界面，而不是深口袋。传统想法会担心：表面界面太平，抗体不好阻断。

VHH 的优势在这里变得明显：

```text
1. 单域结构小，靠近界面时空间约束少；
2. CDR3 往往更长，可以伸到较平的蛋白界面中央；
3. 不需要 VH/VL 配对，计算设计和表达复杂度低；
4. 可做 Fc fusion、二价化、或接入双抗；
5. 对 PPI interface 的 steric competition 更灵活。
```

HR-151 的 IMGT CDR3 长 19 aa，而且含 Cys。这个长 CDR3 很可能是它能形成强 paratope 的重要结构基础。

但这也带来 developability 风险：

```text
长 CDR3 可能带来聚集、异常二硫键、化学不稳定或表达问题；
因此不能只看 blocking IC50，必须同时看 framework health 和 developability。
```

## 8. 不要误读 151：强 blocker 不等于强单药

专利里的动物实验非常重要。

在 A375 melanoma + human PBMC 小鼠皮下移植瘤模型中，专利描述：

```text
anti-PVRIG antibody 151 single-agent group：相对对照没有明显差异；
anti-TIGIT 1708-IgG1 单药、151 + 1708-IgG1 联用、1708-151-IgG1 双抗：肿瘤体积下降；
1708-151-IgG4 双抗：甚至可以完全抑制肿瘤生长。
```

这个结果对我们非常关键：

```text
151 是强 PVRIG-PVRL2 blocker，
但 PVRIG blocker 单药不一定足以产生强体内效应。
```

原因可能包括：

```text
1. PVRIG 与 TIGIT/PD-1 是并行抑制轴；
2. 肿瘤里 CD112/PVRL2 与 CD155/PVR 的表达背景不同；
3. T/NK cell activation 需要多通路解除抑制；
4. VHH 的体内半衰期和价态可能影响药效；
5. Fc format / 装配方式改变功能输出。
```

所以，151 的正确用法不是“直接抄它做单药”，而是：

```text
把它当作强 blocking paratope 证明，
同时研究 format 和 combination 为什么让它变强。
```

## 9. 151 如何变成双抗模块

专利探索了把 anti-PVRIG single-domain antibody 151 通过 linker 接到 anti-TIGIT antibody 1708 上。

核心设计是：

```text
anti-TIGIT antibody 1708
    +
anti-PVRIG VHH 151
    +
GGGGSGGGGS linker
    ↓
1708-151 bispecific antibody
```

151 被接到 1708 的不同位置：

```text
heavy chain N-terminus
heavy chain C-terminus
light chain N-terminus
light chain C-terminus
```

专利观察到：151 的连接方式对 anti-TIGIT antibody 与 TIGIT 的结合基本没有影响。这说明 VHH 模块可以作为相对独立的 PVRIG-binding arm，被挂到 IgG 上。

更重要的是：

```text
1708-151H8 双抗在 MLR 中促进 IFNγ 分泌，
活性强于单用 anti-PVRIG 151H8 或单用 anti-TIGIT 1708。
```

这证明 PVRIG VHH 的成功不只在“它能 bind/block PVRIG”，还在于它可以成为多特异性抗体工程模块。

## 10. 30 和 151 都值得关注：不是只有 151

专利后续还把不同人源化 anti-PVRIG antibodies 接到 anti-TIGIT 1708 重链 N 端，包括：

```text
20H5
30H2
39H2
151H7
151H8
```

其中动物实验显示：

```text
1708-30H2IgG4
1708-151H7IgG4
```

都可以在低剂量下有效抑制肿瘤生长。

这说明：

```text
PVRIG-30 family 也具有真实 format-level 成功潜力；
不能把所有注意力放在 HR-151 / PVRIG-151 上。
```

这点和 2025 年 SHR-2002 的公开摘要方向一致：anti-PVRIG nanobody 可以融合到 anti-TIGIT antibody 的 N 端，构建同时阻断 TIGIT/PVRIG 的 bispecific antibody。

PubMed 摘要显示该类 BsAb：

```text
1. 同时阻断 TIGIT/CD155 和 PVRIG/CD112；
2. T cell activation 增加约 2.8-fold；
3. NK cell cytotoxicity 增加约 1.8-fold；
4. 在 humanized PBMC-reconstituted 和 transgenic mouse models 中有抗肿瘤活性；
5. 食蟹猴 200 mg/kg 重复 4 次未见 dose-limiting toxicity。
```

## 11. 对后续 AI/筛选模型的启发

### 11.1 不要把 HR-151 当作唯一中心

后续模型不能学成：

```text
越像 HR-151 越好
```

正确方向是：

```text
越能用不同 CDR/paratope 复现 PVRIG-PVRL2 blocking，越好。
```

具体应该保留多个正向参考族：

```text
PVRIG-20 family
PVRIG-30 family
PVRIG-38 family
PVRIG-39 family
PVRIG-151 family
```

### 11.2 必须加入 format-aware ranking

VHH 的成功不只由 CDR 决定，还由 format 决定：

```text
naked VHH
VHH-Fc
bivalent VHH
albumin-binding VHH fusion
TIGIT-PVRIG bispecific
```

同一个 PVRIG VHH，在不同 format 中可能有完全不同的体内结果。

### 11.3 评分特征建议

建议后续模型加入：

```text
PVRIG_binding_Kd_prediction
PVRIG_PVRL2_blocking_IC50_prediction
interface_coverage_score
PVRL2_steric_competition_score
CDR3_length_and_shape_score
VHH_framework_health_score
free_cysteine_risk_score
positive_reference_CDR_identity_penalty
VHH_Fc_fusion_compatibility_score
TIGIT_PVRIG_bispecific_compatibility_score
species_cross_reactivity_risk
```

### 11.4 对设计策略的直接建议

```text
1. HR-151/151 作为 positive control 和 leakage exclusion，不作为改几个点的模板。
2. PVRIG-20/30/38/39 也要进入机制参考和相似性排除名单。
3. 第一轮可以先设计裸 VHH 序列，但排序时要提前记录是否适合 Fc fusion / bivalent / bispecific。
4. 不要因为 151 IC50 最强就忽略 30；30 在双抗/format 方向非常值得关注。
5. 对含 Cys 的长 CDR3 要单独做 developability 风险检查。
```

## 12. 和当前结构热点怎么连接

目前没有公开的 PVRIG-VHH 复合物结构，所以不能直接说 151 接触了 R95 或 F139。

但我们可以做机制推断：

```text
这些 VHH 都能阻断 PVRIG-PVRL2，
说明它们的 paratope 必须覆盖、靠近或空间竞争 PVRIG-PVRL2 functional interface。
```

因此在当前结构图谱中，它们应被用来支持：

```text
1. consensus interface 是可被小型 antibody domain 竞争的；
2. R95/H92/R98/W100 和 K135/F139/S143/W144 这些接触带值得保留为 epitope seed；
3. VHH CDR3 的长度/方向可能是形成 steric hindrance 的关键；
4. 但不能把 HR-151 CDR3 直接映射到结构热点，除非未来有 docking/复合物结构/突变证据。
```

这也是为什么现在不该直接上 docking 生成最终 50 条，而应该先把这些真实成功案例变成模型约束。

## 16. 【重要标签】VHH 结合机制与后续 docking 校对标签

这一节是给后续结构预测、复合物 docking、ranking pipeline 做校对用的。它不是最终结论，而是把当前成功案例转成可检查的机制标签。

### 16.1 机制标签总表

| 重要级别 | 标签 | 含义 | 对后续 docking / 结构预测的校对作用 |
| --- | --- | --- | --- |
| HIGH | `BINDER_NOT_ENOUGH` | 只结合 PVRIG 不够，必须能阻断 PVRIG-PVRL2 | docking 结果不能只按 binding energy 排序，必须看是否与 PVRL2 结合位点冲突 |
| HIGH | `BLOCKING_INTERFACE_REQUIRED` | PVRIG-20/30/38/39/151 都有 nM 级 blocking，说明它们命中 functional interface 或其邻近位点 | docking pose 应该靠近 PVRIG-PVRL2 consensus interface，而不是落在远端表面 |
| HIGH | `STERIC_COMPETITION` | VHH 很可能通过空间占位让 PVRL2 不能按原角度贴合 | 需要检查 VHH-PVRIG pose 是否与 PVRL2 产生 clash 或明显重叠 |
| HIGH | `HR151_POSITIVE_CONTROL_NOT_TEMPLATE` | HR-151 是官方阳性和查重对象，不是改几个点的模板 | 结构预测可用 HR-151 校对流程，但设计序列要做 CDR identity 排除 |
| HIGH | `MULTI_FAMILY_POSITIVE_SIGNAL` | 20/30/38/39/151 都有效，不只有 151 | 校对集要保留多个 VHH family，避免模型过拟合 151 |
| MEDIUM | `LONG_CDR3_ADVANTAGE` | HR-151 CDR3 长 19 aa，可能提供突出 paratope | docking 时重点观察 CDR3 是否真正朝向 PVRIG interface，而不是折回 framework |
| MEDIUM | `SHORTER_CDR3_CAN_WORK` | PVRIG-20/30 的 CDR3 较短但也能 block | 不要把 CDR3 越长越好作为硬规则，关键是形状、朝向和界面覆盖 |
| MEDIUM | `CDR3_CYS_RISK_AND_OPPORTUNITY` | HR-151 CDR3 含 Cys，可能稳定构象，也可能带来 developability 风险 | 结构预测要检查 Cys 是否形成合理二硫键或暴露游离 Cys |
| MEDIUM | `FORMAT_DEPENDENCE` | 裸 VHH、VHH-Fc、双抗 format 的药效不同 | docking 只能校对 paratope/interface，不能直接代表体内药效 |
| MEDIUM | `PVRIG30_NOT_SECONDARY` | 30 family 后续双抗结果也有价值 | 校对流程不能只拿 151；30 family 应作为独立成功模式 |
| CAUTION | `NO_PUBLIC_VHH_PVRIG_COMPLEX` | 当前没有公开 PVRIG-HR151 或 PVRIG-151 复合物结构 | 任何“接触 R95/F139/W100”的说法都必须标记为 docking inference，不是实验事实 |
| CAUTION | `NUMBERING_MUST_BE_UNIFIED` | 专利用 Kabat，本地 validator 用 IMGT | docking/paratope 分析前必须统一 ANARCI/IMGT 或明确编号体系 |

### 16.2 后续 docking pose 的最低校对标准

如果后续预测出 VHH-PVRIG 复合物结构，一个 pose 至少要通过这些校对：

```text
1. VHH paratope 应主要由 CDR，尤其 CDR3，接触 PVRIG；
2. VHH 结合位置应靠近 PVRIG-PVRL2 consensus interface；
3. 把 PVRL2 叠回 8X6B/9E6Y 后，VHH 应与 PVRL2 有空间冲突或遮挡；
4. pose 不应只落在 PVRIG 背面、膜远端无关区域或明显非阻断表面；
5. HR-151 / PVRIG-151 可作为 positive control 校对 docking 流程，但不能作为设计模板直接抄；
6. PVRIG-20 / 30 / 38 / 39 / 151 应尽量作为一组校对案例，而不是单点校对；
7. 如果模型预测 VHH 接触 R95/H92/R98/W100 或 K135/F139/S143/W144，应标记为高优先级候选 pose；
8. 如果模型预测只接触 S67 附近但不遮挡 consensus interface，应降低 blocking 可信度；
9. 对含 Cys 的 CDR3，要检查是否有不合理游离 Cys、错误二硫键或暴露疏水 patch；
10. docking score 不能替代 blocking score，只能作为结构解释层证据。
```

### 16.3 推荐的 docking 校对流程

```text
Step 1：预测单体 VHH 结构
        ↓
检查 CDR3 是否伸出、是否塌陷、是否有异常 Cys/疏水暴露
        ↓
Step 2：将 VHH dock 到 PVRIG ECD
        ↓
保留靠近 PVRIG-PVRL2 consensus interface 的 pose
        ↓
Step 3：把 PVRL2 从 8X6B / 9E6Y 叠回 PVRIG
        ↓
计算 VHH 与 PVRL2 的 steric clash / overlap / interface occlusion
        ↓
Step 4：检查 VHH CDR 与 hotspot 的接触
        ↓
重点看 H92/R95/R98/W100 和 K135/F139/S143/W144 区域
        ↓
Step 5：用 20/30/38/39/151 的 known blocking 数据做流程校对
        ↓
如果强 blocker 的预测 pose 完全不遮挡 PVRL2，需要反查 docking 参数或抗原构象
```

### 16.4 对 docking 结果的解释分级

建议后续所有 VHH-PVRIG docking 结果都按下面分级，不要只给一个总分：

| 等级 | 判定 | 解释 |
| --- | --- | --- |
| A | CDR3/CDR 主导接触 + 靠近 consensus interface + 明显遮挡 PVRL2 | 最符合 VHH blocker 机制，可进入后续精修 |
| B | 靠近 interface，但遮挡 PVRL2 不充分 | 可能是 binder 或弱 blocker，需要重新采样或改造 CDR |
| C | 接触 PVRIG 表面但远离 PVRL2 interface | 可能只是 binder，不应作为 blocking 优先候选 |
| D | pose 主要靠 framework 非特异接触或结构不合理 | 应丢弃或重新预测 |
| X | 与 HR-151/Tab5/公开阳性 CDR 过相似 | 即使 pose 好，也应作为 leakage risk 排除 |

### 16.5 最重要的校对原则

> 对这组 VHH 来说，后续结构预测和 docking 的核心校对问题不是“它能不能贴上 PVRIG”，而是“它贴上的位置和角度，能不能让 PVRL2 贴不上来”。

因此 Case 02 后续作为流程校对时，应优先验证：

```text
known strong blockers
        ↓
是否被 docking 流程排到 PVRIG-PVRL2 interface 附近
        ↓
是否产生 PVRL2 steric occlusion
        ↓
是否由 CDR3/CDR 主导接触
```

如果流程不能把这些已知强 blocker 解释为 interface-occluding binders，那么这个 docking 流程不应该直接用于筛最终候选。
